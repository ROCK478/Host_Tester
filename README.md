# bench.py — тестирование доступности HTTP-серверов

Консольная программа на Python для проверки доступности серверов по HTTP(S) и
замера времени ответа. Запросы к разным хостам, а также несколько запросов к
одному хосту, выполняются **конкурентно** (асинхронно, на базе `asyncio` +
`httpx`), поэтому даже большое количество запросов выполняется быстро.

## Установка

Требуется Python 3.8+.

```bash
pip install -r requirements.txt
```

## Запуск

### Базовый пример

```bash
python bench.py -H https://ya.ru,https://google.com -C 5
```

### Аргументы командной строки

| Ключ | Короткий | Обязательный | Описание |
|---|---|---|---|
| `--hosts` | `-H` | да* | Список хостов через запятую без пробелов, напр. `https://ya.ru,https://google.com` |
| `--file` | `-F` | да* | Путь до файла со списком адресов (по одному на строку) |
| `--count` | `-C` | нет | Количество запросов на каждый хост. По умолчанию `1` |
| `--output` | `-O` | нет | Путь до файла для сохранения отчёта. Если не указан — вывод в консоль |

\* Обязателен ровно один из ключей `-H` или `-F` — указать оба одновременно нельзя.

Адреса должны соответствовать формату `https://example.com` (со схемой
`http://` или `https://`), иначе программа выведет понятную ошибку и
завершится, не выполняя запросов.

### Пример с файлом адресов

`hosts.txt`:
```
https://ya.ru
https://google.com
```

```bash
python bench.py -F hosts.txt -C 5 -O result.txt
```

## Формат вывода

Для каждого хоста выводится отдельный блок статистики:

```
==================================================
Host:    https://ya.ru
Success: 5
Failed:  0
Errors:  0
Min:     0.0512 с
Max:     0.0981 с
Avg:     0.0703 с
==================================================
Host:    https://google.com
Success: 4
Failed:  1
Errors:  0
Min:     0.0421 с
Max:     0.1204 с
Avg:     0.0733 с
==================================================
```

Где:
- **Host** — имя тестируемого хоста;
- **Success** — количество запросов, завершившихся успешно (код ответа < 400);
- **Failed** — количество запросов, на которые сервер ответил ошибкой (код 4xx или 5xx);
- **Errors** — количество запросов, во время которых сервер был недоступен
  (таймаут, обрыв соединения, ошибка DNS и т.п.);
- **Min / Max / Avg** — минимальное, максимальное и среднее время выполнения
  запроса среди тех запросов, на которые был получен ответ от сервера
  (Success + Failed). Если ответов не было ни разу, выводится `н/д`.

## Обработка ошибок

Все ожидаемые ошибки (некорректный формат хоста, нечисловое значение
`--count`, отсутствующий файл, недоступность сервера и т.п.) перехватываются
через `try/except` и выводятся пользователю в виде понятного текстового
сообщения, без "сырых" traceback'ов Python.

## Конкурентность

Запросы выполняются асинхронно с помощью `asyncio` и `httpx.AsyncClient`:
- все хосты обрабатываются параллельно;
- все `--count` запросов к одному хосту также отправляются параллельно.

Это значительно ускоряет тестирование при большом количестве хостов и/или
запросов по сравнению с последовательным выполнением.
